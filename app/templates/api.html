User api

/api/user?id=xxx,/api/user?nickname=xxx

You can access our user api in the format above.

Both id and nickname can be used to locate certain user, but if you use them in the same time, we will ignore the nickname argument.

The response is a json string. If you want to know more detailed,just try.

Blogs api

/api/blogs?author=xxx

Blogs api use author's nickname to locate a resource and will return all the blogs writen by it but not in detail(comments are not included)

However, it will return the blog's id, you can access our blog_page api to access detailed information.

Our api response is always a json string.

Blog_page api

/api/blog_page?id=xxx

Blog_page api use id to locate a certain blog and will return detailed information including both its comments and sub_comments.

It is not easy to access a blog's id, so we suggest to use this api with blogs api.

For sure, this api also respond with a json string.
