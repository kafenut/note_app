1.建立仓库
仓库英文为repository，也可称之为版本库。其本质可理解为一个文件夹，如果你指定了一个文件夹建立git仓库，我们就能用git管理这个文件夹下的文件了。
首先我们先在命令行中，把当前路径移动到我们想管理的文件夹的位置，之后在命令行中键入：

git init

git会在目标文件夹中建立一个名为.git的隐藏文件夹，这个隐藏文件夹也是文件夹是git仓库的标志。之后你就能够用git管理目标文件夹（这个文件夹被称为工作区）中的文件了。

2.添加文件
git并不会自动追踪工作区中的所有文件的变化，要让git追踪文件变化，首先要把文件添加进仓库中。命令行中键入：

git add file

即可把文件添加进git仓库之中，不过add命令仅仅只是把文件放入缓存区中，要把文件提交到仓库中，则需要再在命令行中键入提交命令：

git commit -m "first commit message"

-m命令后的字符串为本次commit的描述，为了他人与自己之后的阅读体验，最好每一次commit都要加上恰当的描述。
commit完成后，文件即添加进了git仓库。commit命令是将缓存区中的文件全部添加进仓库，所以在一次commit命令之前，我们可以多次使用add命令，这样就可以做到一次添加多个文件进仓库。

除了提交单个文件之外，git add命令还有其他的几个添加格式
git add *    ——添加所有文件
git add *.txt ——只添加文件夹内所有的txt文件
git add folder/   ——只添加folder文件夹中的文件
当然你也能够把上面的添加方式组合起来：
git add folder/*.txt  ——只添加folder文件夹中的txt文件

如果要撤销add，可在命令行中键入：
git rm -cach file
当然rm命令也能用上面的格式使用，不过如果要撤销所有add，还需要添加-r选项，即git rm -cach -r *。

我们可以随时使用git status查看当前路径下的git仓库的状态，他会显示git仓库add与commit状态，也可以查看还有哪些文件没有添加进仓库。

3.移除文件
rm命令本职是把文件移除仓库：

git rm file 

由于把文件移除仓库也是一次修改（可以视为一次add），所以执行rm命令后还需要执行一次commit命令。
