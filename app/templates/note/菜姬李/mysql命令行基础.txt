事先说明，以下操作都是在mysql 8.0.17下进行的。

零、关闭/开启mysql服务（windows）
mysql是作为系统的一个网络服务存在的，可以通过远程端口连接mysql服务器。在使用mysql前需要先确保mysql服务正在运行当中
net start mysql服务名称 (默认为名称mysql，开在3306端口，我的服务器是mysql3306)
net stop mysql服务名称 
tips：需要用管理员模式运行该命令，可用ctrl+q，搜索cmd，再用ctrl+shift+enter进入管理员模式；或直接在开始菜单中找到管理员模式运行cmd。

一、命令行准备工作
要在命令行中使用mysql，首先需要将添加进系统环境变量中，或者你自己将当前目录移至mysql的所在，一般在安装时mysql便已经询问了是否将mysql添加进系统变量中。在命令行中输入mysql回车，即可查看mysql是否在环境变量中
如若不在，可通过右键桌面此电脑——属性——高级系统设置——环境变量，编辑系统变量中的path项，新建一个路径并把mysql.exe所在的路径添加进去（mysqlserver\bin）

二、链接数据库
1.本地连接——mysql -u 用户名 -p
键入enter后，会提示输入密码，之后即进入mysql命令行模式
tips：对刚创建的mysql服务器，root用户是没有密码的，可直接进入mysql命令行模式，后续再设置密码
2.远程连接——mysql -h ip地址 -u 用户名 -p
其中ip地址表示远程服务器的地址，而非发起连接请求的主机ip。键入enter后，会提示输入密码，之后即进入mysql命令行模式（待考证）
tips：mysql 登录需三个参量，host（ip地址），user和passwd，mysql会储存用户所对应的ip（一般默认是本机地址localhost），即便用户名和密码都正确，如果host错误，也无法连接mysql。要想远程连接，则需要改动mysql服务中的，名为mysql数据库，设置其中user表对应用户的host值为%（所有ip都可以访问），具体操作我会在其他笔记中详细描述
顺道一提，用户的host是无法指定多个ip的，要实现指定多个ip，只能通过系统的防火墙实现（待考证）
3.退出命令行模式——exit

以下操作均在mysql命令行模式运行，大部分语句后面都需加 “ ; ” ，分号为执行mysql语句的开始标志
三、修改密码
mysql修改密码的方法有很多种，我这里只介绍一种
1.mysql命令行模式修改密码——set password for 用户名@用户host = ’新密码’
需要注意的是，只有root用户能修改所有用户的密码（除非你给其他用户访问mysql库的权限），其他用户只能修改自身的密码（废话）

四、创建删除用户，并给用户授权
1.创建用户——create user '用户名'@'用户host' identified by '密码'
创建用户，并指定其密码与可发起访问的ip地址
2.用户赋权——grant select,delete,update,insert,create,drop,alter(或all) on 数据库名.* to ”用户名”@"用户host"  (with grant option);
把指定数据库的指定权限，赋给指定用户，注意数据库名后的“.*”不可省略。若指定with grant option，则该用户也可将其权限赋给其他用户。
3.删除用户——drop user 用户名@用户host;
删除指定用户

五、简单数据库操作
1.显示所有数据库——show databases;
2.创建数据库——create database 数据库名;
3.删除数据库——drop database 数据库名;
4.连接(切换)数据库——use 数据库名;
在对一个数据库进行操作前，需要先连接数据库

六、数据库备份与恢复
1.数据库备份——mysqldump -u 用户名 -p [-h 目标主机host --port 目标主机端口] dbname > 备份文件名.sql
利用mysqldump进行数据库备份（备份为一系列的sql语句），指定-h --port可以备份远端数据库，不指定即备份本地的数据库。
2.数据库恢复
数据库恢复介绍两种方法：
a.命令行备份——mysql -u root -p dbname < sql文件路径
b.mysql命令行模式中备份——source sql文件路径
